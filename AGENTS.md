# Expo HAS CHANGED

This project is on **Expo SDK 56** (upgraded from SDK 55 on 2026-07-13). Read the exact versioned docs at https://docs.expo.dev/versions/v56.0.0/ before writing any code.

When adding native modules, always use `npx expo install <pkg>` (never plain `npm install`) so versions stay pinned to SDK 56 — mixing in mismatched-SDK packages causes native build failures (e.g. unresolved symbols like `OptimizedRecord`). If `app.json`'s `sdkVersion` field and the installed `expo` package version ever drift apart, `expo install --fix` silently resolves against the *declared* `sdkVersion`, not the installed package — keep them in sync.

`@expo/ui` is now **stable** as of SDK 56. Its SwiftUI `TextField` is properly controlled via `useNativeState` + `text`/`onTextChange` (an `ObservableState<string>`) instead of the old `defaultValue`-only API — see [LiquidTextField.tsx](src/components/common/LiquidTextField.tsx) for the pattern. `expo-font` and `expo-splash-screen` are now required explicit dependencies (previously implicit) — `@expo/vector-icons` needs `expo-font` as a peer, and the top-level `splash` key in `app.json` was removed in favor of the `expo-splash-screen` config plugin. RN 0.85 also dropped `StyleSheet.absoluteFillObject` in favor of `StyleSheet.absoluteFill`.

`@gorhom/bottom-sheet` is fully removed — `LiquidBottomSheet.tsx` already covers every sheet in the app via `@expo/ui`'s native SwiftUI/Jetpack Compose sheets, and the old `AppBottomSheet.tsx`/`BottomSheetModalProvider` were dead weight. `LiquidHeaderMenu.tsx`'s Android path was also migrated off `react-native-paper`'s JS `Menu` onto `@expo/ui/community/menu`'s `MenuView`, giving a real native Compose `DropdownMenu` there instead (iOS was already native SwiftUI `Menu` and is unchanged). `LiquidTabs.tsx` was deliberately **not** migrated to `@expo/ui/community/segmented-control` — that component has no per-segment icon slot, and the Android tab bar (`UdhaarScreen`, others) relies on icons; the custom pill-tab implementation stays.
